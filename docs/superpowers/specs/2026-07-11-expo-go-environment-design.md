# Cucoudle: окружение Expo Go для реального iPhone

## Цель

Подготовить репозиторий Cucoudle к локальной разработке React Native-приложения на Mac и запуску текущей версии приложения на реальном iPhone через Expo Go.

## Выбранный подход

Проект создаётся на базе актуального стабильного Expo SDK с TypeScript. Для первого запуска используется Expo Go: разработчик запускает Metro на Mac, а приложение на iPhone подключается к нему по QR-коду. Собственная нативная сборка, CocoaPods, сертификаты Apple Developer и EAS на этом этапе не требуются.

## Структура и зависимости

- Expo-проект размещается непосредственно в корне существующего репозитория.
- Используется минимальный официальный шаблон TypeScript без добавления продуктовых функций.
- Версии React Native, React и связанных пакетов определяются выбранным Expo SDK и не задаются вручную независимо от него.
- В качестве менеджера пакетов используется npm; в репозиторий входят конфигурация Expo, TypeScript, npm-скрипты и `package-lock.json`.
- Сгенерированные зависимости, локальные данные Expo, системные файлы и секреты исключаются из Git.

## Запуск на устройстве

Mac и iPhone должны иметь сетевую доступность друг к другу. Разработчик устанавливает Expo Go из App Store, запускает проект командой `npm start` и сканирует QR-код камерой iPhone или внутри Expo Go. Если локальная сеть блокирует прямое подключение, используется `npm run start -- --tunnel`.

## Обработка проблем

- Проверка окружения должна выявлять несовместимые версии Node.js и зависимостей Expo.
- Если LAN-подключение не работает, документация предлагает tunnel-режим как резервный путь.
- Если функциональность в будущем потребует неподдерживаемый Expo Go нативный модуль, проект должен перейти на Expo development build; этот переход не входит в текущий объём.

## Проверка результата

Настройка считается готовой, если:

- зависимости устанавливаются без ошибок;
- диагностика Expo не обнаруживает проблем конфигурации или несовместимых пакетов;
- TypeScript-проверка завершается успешно;
- Metro запускается и выдаёт QR-код/адрес проекта для Expo Go;
- README содержит короткие инструкции запуска на реальном iPhone и резервную команду для tunnel-режима.

Фактическое открытие приложения на iPhone требует действия пользователя на устройстве и проверяется совместно после подготовки локальной части.
