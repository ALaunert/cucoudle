# Дизайн питч-презентации Cucoudle (хакатон)

## Цель

Собрать презентацию для защиты проекта Cucoudle перед жюри хакатона: короткий
demo-driven питч (3–5 минут, ~8 слайдов). Презентация делается заранее, до
демо-дня, поэтому центральный демо-слайд — заглушка под живой показ/видео.

## Формат и аудитория

- **Формат:** HTML-слайды, публикуются как Artifact на claude.ai (открываются с
  любого экрана, показываются проектором).
- **Аудитория:** жюри хакатона.
- **Длительность:** 3–5 минут, demo-driven нарратив.
- **Язык:** русский.

## Визуальная айдентика

Источник — логотип проекта (облако-«спич-бабл» с `< >` и смайлом).

- почти чёрный фон (тёмная тема — уместно для dev-инструмента);
- акцентный градиент **фиолетовый → голубой** (violet → cyan), как в лого и
  вордмарке;
- дружелюбные скруглённые формы, «карточки» под контент;
- моноширинный шрифт для команд/кода, гротеск для заголовков;
- тэглайн: **«AI coding agents. One chat.»**
- лёгкие анимации появления; дизайн корректен и в light-теме.

Все цвета — через CSS-переменные, чтобы перекрасить одним движением (дизайн ещё
дорабатывается).

## Структура (8 слайдов)

1. **Титул** — Cucoudle, тэглайн, состав команды: Кирилл Богачев, Алексей
   Лаунерт, Сергей Александров.
2. **О чём проект** — суть в одном экране + один акцент на боль: агент ждёт
   твоего ответа или упал, а ты не за компьютером; агентские CLI-сессии заперты
   на рабочем месте.
3. **Демо** — полноэкранная заглушка «Demo»: место под видео/живой показ,
   минимум текста (выступающий уходит с презентации на живую демонстрацию).
4. **Почему мы** — преимущества против официальных приложений:
   - единый пульт к Claude/Codex/Cursor — а не отдельный чат-бот под каждого;
   - официальные мобильные приложения агентов не управляют *твоей* запущенной
     CLI-сессией на компьютере — Cucoudle управляет;
   - ничего не меняешь в привычке: та же команда `claude`/`codex`;
   - fallback обязателен — без демона shim просто запускает настоящий бинарь,
     установка не ломает CLI.
5. **Как всё работает** — простая диаграмма без глубокой техники:
   терминал → компьютер (daemon/PTY) → облако (relay) → телефон; ввод с телефона
   возвращается в реальную сессию.
6. **Что уже работает** — перечень готового и проверенного, помечен «будет
   доработано»: relay + общий протокол, desktop-daemon с PTY-мостом и
   shim'ами, кросс-язык live-smoke desktop↔relay↔mobile, deployment-bundle;
   упомянуть покрытие тестами.
7. **Что можно докинуть** — roadmap: мобильное Expo-приложение Action Inbox,
   structured interactions (Approve/Reject/choices), capability negotiation,
   push-уведомления, device-auth и шифрование, автозапуск демона.
8. **Финал** — тэглайн + призыв/контакт.

## Границы честности (важно для жюри)

Слайд 6 «что работает» отделён от «target»-возможностей (слайд 7). Мобильного
Expo-приложения ещё нет — на слайде 6 не заявляем его готовым. Реально работает
техническая часть (relay + daemon + PTY + кросс-язык smoke), её и показываем как
подтверждённую.

## Не входит

- Реальные скриншоты мобильного приложения (его ещё нет) — на демо-слайде
  плейсхолдер под будущее видео/показ.
- Точные роли участников в команде (при желании добавим позже).
- Экспорт в PDF/PPTX — при необходимости печатается из браузера.

## Технические заметки по сборке

- один самодостаточный HTML-файл (Artifact): весь CSS/JS инлайн, без внешних
  ресурсов (строгий CSP);
- навигация по слайдам стрелками/кликом; счётчик слайдов;
- адаптивность: контент не ломает горизонтальный скролл body;
- логотип встраивается data-URI при наличии файла, иначе — CSS-реконструкция
  облака-спич-бабла.
