# Cucoudle: единые инструкции и журнал разработки

## Цель

Обеспечить Codex и Claude Code одинаковыми командными инструкциями и накапливать проверенный материал о ходе и результате разработки для итоговой презентации хакатона.

## Единый источник инструкций

- Корневой `AGENTS.md` содержит общие правила работы AI-агентов.
- Корневой `CLAUDE.md` импортирует `AGENTS.md` и не дублирует его содержимое.
- Оба файла входят в Git, поэтому все разработчики и агентские сессии получают одинаковые правила.

## Документы проекта

### `docs/PROGRESS.md`

Хронологический append-only журнал значимых инкрементов. Каждая запись содержит дату, цель инкремента, выполненную работу, затронутые компоненты, проверки, решения и проблемы, а также следующий шаг. Старые записи не переписываются.

### `docs/FINAL_IMPLEMENTATION.md`

Актуальный цельный снимок продукта. После каждого значимого инкремента документ обновляется, а не дополняется отдельным журналом. Он описывает проблему, ценность, пользовательские сценарии, реализованные функции, архитектуру и стек, ограничения и готовый сценарий демонстрации.

## Правило завершения инкремента

После каждого значимого завершённого изменения агент обязан:

1. добавить запись в `docs/PROGRESS.md`;
2. привести `docs/FINAL_IMPLEMENTATION.md` в соответствие с фактическим состоянием проекта;
3. указать реально выполненные проверки;
4. не считать инкремент завершённым и не создавать его коммит до обновления обоих документов.

В документы заносятся только подтверждённые факты. Планы и предположения явно отделяются от реализованного состояния.

## Начальное состояние

Оба документа создаются с начальным содержимым на основе README и существующих спецификаций. На текущем этапе подтверждено назначение Cucoudle как мобильного приложения для удалённого управления сессиями Cursor, Codex и Claude; продуктовая реализация ещё не зафиксирована в репозитории.

## Проверка

- `CLAUDE.md` содержит корректный импорт `@AGENTS.md`.
- `AGENTS.md` однозначно задаёт момент и состав обновления обоих документов.
- Журнал прогресса имеет шаблон append-only записи.
- Финальное описание отделяет реализованные возможности от планов.
- Все четыре рабочих файла и эта спецификация проходят `git diff --check`.
